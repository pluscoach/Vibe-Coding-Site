# Phase 2 작업 지시서 — 헤더 + 히어로 + 마지막 CTA

## 0. 이번 Phase의 의미

Phase 2는 단순히 두 섹션을 채우는 작업이 아니다.

이번 Phase에서 만드는 것:
- **헤더의 톤** → Phase 3~6 내내 페이지 상단에 고정 노출
- **히어로의 디자인 언어** → 전체 페이지의 시각적 첫인상
- **CTA 버튼의 형태** → 페이지에 5~7번 반복 등장할 디자인 원본

즉, **Phase 2 결과물이 곧 사이트 전체의 디자인 기준**이 된다. 폰트 크기·여백·버튼 색상·호버 효과·반응형 분기점이 여기서 확정되고, Phase 3~6은 이걸 따른다.

→ **빨리 만들기보다, 정확하게 만들기.** 이 Phase는 시간이 좀 걸려도 된다.

## 2. 상단 sticky 미니헤더

### 2.1 구조
```
┌──────────────────────────────────────────────────┐
│ A.MAKERS                            [1기 신청]   │
└──────────────────────────────────────────────────┘
```

### 2.2 명세
- **위치:** `sticky top-0 z-50`
- **배경:** `bg-white/80 backdrop-blur` (Stripe·Linear 톤)
- **하단 보더:** `border-b border-slate-200` (스크롤 시 본문과 분리)
- **높이:** 데스크탑 64px (`h-16`), 모바일 56px (`h-14`)
- **컨테이너:** `max-w-6xl mx-auto px-6 md:px-8`, 내부 `flex items-center justify-between`

### 2.3 좌측 — 로고
- 텍스트: **`A.MAKERS`** (대문자 + 점)
- 폰트: `font-bold text-lg md:text-xl tracking-tight`
- 색상: `text-slate-900`
- 호버: `hover:text-slate-700` (미세하게)
- 링크: `href="#hero"` (클릭 시 페이지 최상단)

### 2.4 우측 — CTA 버튼
- 텍스트: **`1기 신청`**
- 크기: `px-4 py-2 text-sm md:text-base` (히어로 메인 CTA보다 작게)
- 색상: `bg-emerald-600 hover:bg-emerald-700 text-white`
- 모서리: `rounded-lg`
- 폰트: `font-semibold`
- 링크: `href="#final-cta"`

### 2.5 모바일 처리
- 햄버거 메뉴 만들지 않는다. 1페이지 랜딩이라 불필요.
- 좌측 로고 + 우측 CTA만 유지.
- 화면 폭 좁아져도 두 요소 다 노출되어야 함.

## 3. 섹션 1 — 히어로

### 3.1 레이아웃 (데스크탑)
```

```

│  │ 비주얼 영역     │    │
│  └─────────────────┘    │
└─────────────────────────┘

### 3.3 카피 (확정, 수정 금지)

**헤드라인:**
```
4주 안에 AI로
당신의 첫 서비스를
출시합니다
```
- 폰트: `text-5xl md:text-6xl lg:text-7xl`
- 굵기: `font-bold` (700) ~ `font-extrabold` (800)
- 줄 간격: `leading-tight`
- 색상: `text-slate-900`
- 줄바꿈: 위 표기대로 3줄 (모바일에서는 자연 줄바꿈 허용)

**서브헤드 (헤드 아래):**
```
개발 경험이 없어도 괜찮습니다.
시장 검증부터 MVP 제작, 출시까지 1:1로 함께 진행합니다.
```
- 폰트: `text-lg md:text-xl`
- 굵기: `font-normal` ~ `font-medium`
- 색상: `text-slate-600`
- 헤드와의 간격: `mt-6`

**CTA 버튼:**
```
1기 무료 상담 신청
```
- 크기: `px-8 py-4 text-lg`
- 색상: `bg-emerald-600 hover:bg-emerald-700 text-white`
- 모서리: `rounded-xl`
- 폰트: `font-semibold`
- 링크: `href="#final-cta"`
- 서브헤드와의 간격: `mt-10`
- 인라인 블록으로 (텍스트 길이만큼만 차지)
- 호버 시 미세한 `transition` + `shadow-lg` 정도

**CTA 아래 미세 정보 (3줄):**
```
· 1기 한정 5명 · 정가 100만원 → 50만원
· 1주차 불만족 시 전액 환불
```
- 폰트: `text-sm`
- 색상: `text-slate-500`
- CTA와의 간격: `mt-6`
- 각 줄 간격: `space-y-1`

### 3.4 우측 비주얼 영역

이번 Phase에서는 **실제 이미지 없음.** Phase 3에서 결과물 갤러리 들어올 때 처리.

지금 처리:
- 빈 박스 자리만 잡기
- 배경: `bg-slate-100` 또는 `bg-gradient-to-br from-slate-100 to-slate-200`
- 모서리: `rounded-2xl`
- 종횡비: `aspect-square` 또는 `aspect-[4/3]`
- 내부에 작은 캡션 (검수자만 볼 임시 표시): "비주얼 영역 — Phase 3에서 결과물 모자이크 또는 페르소나 시스템 스크린샷 들어올 자리"
- 캡션은 `text-slate-400 text-sm`

### 3.5 섹션 전체 패딩
- 데스크탑: `py-24 md:py-32 lg:py-40`
- 컨테이너: `max-w-6xl mx-auto px-6 md:px-8`

### 3.6 sticky 헤더와의 간격
헤더가 sticky로 떠 있으므로, 히어로 상단에 헤더 높이만큼 추가 패딩 필요 없음 (sticky는 자기 자리 차지하므로). 단, 페이지 최초 진입 시 헤더 아래로 히어로가 자연스럽게 시작되어야 함 — `<body>` 직후 헤더, 헤더 직후 `<main>` 안에 `<section id="hero">`.

## 5. CTA 버튼 디자인 — 재사용 컴포넌트화

히어로·헤더·마지막 CTA 세 곳에 등장하는 버튼은 **시각적으로 일관**되어야 한다.

### 5.1 버튼 변형 3종
| 위치 | 크기 | 클래스 |
|---|---|---|
| 헤더 (작은 CTA) | Small | `px-4 py-2 text-sm md:text-base font-semibold rounded-lg bg-emerald-600 hover:bg-emerald-700 text-white transition` |
| 히어로 (메인 CTA) | Large | `px-8 py-4 text-lg font-semibold rounded-xl bg-emerald-600 hover:bg-emerald-700 text-white transition hover:shadow-lg` |
| 마지막 CTA (네이비 위) | Large + 강조 | `px-8 py-4 text-lg font-semibold rounded-xl bg-emerald-600 hover:bg-emerald-700 text-white transition shadow-xl` |

### 5.2 일관성 원칙
- 색상: 무조건 `emerald-600` → `emerald-700` 호버
- 모서리: `rounded-lg` (small) 또는 `rounded-xl` (large)
- 폰트 굵기: `font-semibold` 통일
- 트랜지션: 모든 버튼에 `transition` 적용

이 패턴은 Phase 3~6에서 추가되는 모든 CTA 버튼에도 적용. 헌법 §3.3 컬러 시스템의 일부로 간주.

## 7. 반응형 검증

다음 너비에서 점검:

| 너비 | 점검 사항 |
|---|---|
| 375px (모바일) | 헤더 양쪽 요소 노출, 히어로 1칼럼, 헤드라인 한 줄에 안 들어가도 OK (자연 줄바꿈), CTA 버튼 누르기 편한 크기, 가로 스크롤 없음 |
| 768px (태블릿) | 헤더 정상, 히어로 1칼럼 유지 또는 2칼럼 시작 (선택), 마지막 CTA 중앙 정렬 깔끔 |
| 1280px (데스크탑) | 히어로 2칼럼 명확, 좌측 카피 + 우측 비주얼 박스 균형, 마지막 CTA 가운데 정렬 + 양 옆 충분한 여백 |

**히어로의 2칼럼 분기점:** `md:` (768px) 또는 `lg:` (1024px). 둘 중 어디서 시작할지는 Claude Code가 디자인 톤 보고 판단. 권장은 `lg:` (1024px) — 그래야 태블릿에서도 1칼럼으로 보여 카피가 더 강조됨.
