# PROJECT.md — 프로젝트 헌법

> 이 문서는 **이 프로젝트가 무엇이고, 왜 만들고, 어떻게 의사결정해야 하는지**의
> 단일 기준점이다. 작업 중 판단이 갈릴 때마다 이 문서로 돌아온다.
>
> Phase 지시서가 "이번 주 할 일"이라면, 이 문서는 "프로젝트 헌법"이다.
> 헌법과 Phase 지시서가 충돌하면 **헌법이 우선**한다.

---

## 1. 프로젝트 정체성

### 1.1 한 줄 정의
**"비전공자 1인 사업가가 4주 1:1 컨설팅으로 AI 서비스를 출시하게 만드는 모집 랜딩페이지."**

이 한 줄에서 벗어나는 모든 작업은 보류 대상이다.

### 1.2 이 사이트가 아닌 것
다음 중 하나라도 해당된다면 이 사이트는 그것이 **아니다**:

- ❌ 강의 플랫폼 (탈잉·클래스101 같은 종합 플랫폼)
- ❌ 전자책 판매 사이트
- ❌ 결제·로그인·회원관리 시스템
- ❌ 블로그·SEO 콘텐츠 허브
- ❌ AI 도구 소개 사이트
- ❌ "AI 잘 쓰는 법" 콘텐츠 사이트

이 모든 가능성은 6개월 이후로 미뤘다. 지금은 **1기 5명 모집을 위한 단일 페이지**만이 존재 이유.

### 1.3 사이트의 단 하나의 목적
> **"무료 상담 신청 폼 1건 제출"**

방문자에게 시키고 싶은 행동은 이것 하나. 다른 모든 콘텐츠·디자인·카피는 이 행동을 끌어내기 위한 도구.

## 2. 비즈니스 컨텍스트

### 2.1 운영자
- **플러스코치** — 1인 사업자, 비전공자 출신
- 사업소득 연 5,000만원 규모
- 이미 운영 중인 다른 사업: 자동매매 프로그램 판매, BlogAutoFriends SaaS, 유튜브 채널 2.5만 등

### 2.2 강의 상품 정의
| 항목 | 내용 |
|---|---|
| 상품명 | 바이브 코딩 1:1 컨설팅 |
| 형식 | 4주 온라인 1:1 (줌) |
| 1주차 | 시장 검증 — 100만 데이터로 페르소나 50명 추출 |
| 2주차 | AI 기획 — 좋은 질문법 + 1,000만원 수익 설계 시스템 |
| 3주차 | MVP 제작 — 실제 동작하는 서비스 |
| 4주차 | 출시 — 광고 집행 + 첫 결제 시도 |
| 1기 모집 인원 | 5명 한정 |
| 1기 가격 | 50만원 (정가 100만원) |
| 환불 보장 | 1주차 후 불만족 시 전액 환불 |

### 2.3 타겟 페르소나 (구체적으로 명시)
1. 본업 있는 직장인, 부수입 만들고 싶음
2. AI에 관심 많은데 뭘 만들지 모름
3. 사업 아이템 있는데 만들 줄 모르는 사장
4. 코딩 강의 들었는데 결과물 못 만든 사람

타겟 외 사용자(예: 깊이 있는 코딩 학습자, 즉시 큰돈 벌고 싶은 사람)는 **의도적으로 거른다.** 거르는 것 자체가 전환율 상승 도구.

### 2.4 차별점 (3가지, 이게 전부)
1. **100만 데이터 페르소나 분석 시스템** — 자체 제작, 다른 어디에도 없음
2. **1:1 진행** — 녹화 강의·단톡방 강의 아님
3. **출시까지 함께** — 만드는 것뿐 아니라 광고·결제 시도까지

이 3개로 차별점은 충분. 더 추가하지 않는다. 무기는 많을수록 약해진다.

## 3. 디자인 헌법

### 3.1 톤 한 줄
> **"GPT 강의 사이트가 아니라, Y Combinator에 지원하는 예비 창업자를 위한
> SaaS 액셀러레이터 랜딩페이지."**

이 한 줄을 누군가에게 설명할 수 없다면 디자인 결정을 잠시 멈춰라.

### 3.2 레퍼런스
- **Linear** (linear.app) — 미니멀·신뢰감
- **Stripe** (stripe.com) — 정보 밀도·여백 운영
- **Notion** (notion.so) — 친근하면서도 단정함

이 세 사이트의 공통점: **여백 크고, 컬러 적고, 카피 짧고, 폰트 단정함.**

### 3.3 컬러 시스템 (고정, 변경 금지)
| 용도 | 컬러 | Tailwind |
|---|---|---|
| 기본 배경 | 화이트 | `bg-white` |
| 보조 배경 | #F8FAFC | `bg-slate-50` |
| 강조 배경 | #0F172A | `bg-slate-900` |
| 본문 텍스트 | #0F172A | `text-slate-900` |
| 보조 텍스트 | #475569 | `text-slate-600` |
| 포인트 컬러 | 민트 | `bg-emerald-500/600` |
| 위험 표시 | 레드 | `text-red-500` |

**이 7개 외 컬러 사용 금지.** 추가하고 싶을 때마다 이 헌법으로 돌아온다.

### 3.4 타이포 시스템
- 폰트: **Pretendard 단일.** 다른 폰트 도입 금지
- 헤드라인: 56~64px, weight 700~800
- 서브헤드: 24~28px, weight 500
- 본문: 16~18px, weight 400
- 캡션: 14px, weight 400

### 3.5 디자인 금지 사항 (자주 어기게 되는 것들)
- ❌ 이모지 사용 (체크마크 ✓ SVG는 OK)
- ❌ 과한 그라데이션 (1~2군데 포인트만)
- ❌ 진한 그림자 (`shadow-md` 이하)
- ❌ 둥근 모서리 과용 (`rounded-2xl` 기본)
- ❌ "수강신청!" 같은 흥분 톤
- ❌ 네온·형광 컬러
- ❌ 애니메이션 과용 (정적 신뢰감 우선)
- ❌ 자동 재생 영상·자동 캐러셀
- ❌ 팝업·플로팅 배너 (CTA는 자연스럽게 섹션 안에)

### 3.6 디자인 결정 기준
판단 흐들릴 때 다음 질문을 던진다:
1. **Stripe가 이렇게 만들까?** — Yes면 OK
2. **이거 빼도 핵심 정보 전달되나?** — Yes면 빼라 (덜어내기 우선)
3. **모바일에서도 깔끔한가?** — No면 다시
4. **눈이 어디에 먼저 가는가?** — CTA·핵심 카피여야 한다

## 4. 기술 헌법

### 4.1 기술 스택 (고정)
- **HTML/CSS/JS 정적 1페이지**
- **Tailwind CSS** (CDN, 빌드 없음)
- **Pretendard 폰트** (CDN)
- **이외 일체의 프레임워크·번들러·패키지 금지** (Phase 6까지)

### 4.2 파일 구조 (변경 시 헌법 개정 필요)
```
/
├── index.html              # 메인 랜딩 (모든 섹션 한 파일)
├── style-guide.html        # 디자인 토큰 검증 (Phase 1 산출물)
├── thanks.html             # 신청 완료 페이지 (Phase 6에서 추가)
├── assets/
│   ├── images/             # 결과물 스크린샷, 강사 사진 등
│   └── og/                 # OG 이미지, favicon
├── PROJECT.md              # 이 문서
├── phaseN-instructions.md  # 각 Phase 지시서
├── README.md               # 프로젝트 설명, Phase 진행 상태
└── .gitignore
```

### 4.3 기술 결정 기준
판단 흔들릴 때 다음 질문:
1. **빌드 도구 없이 가능한가?** — No면 보류
2. **이 패키지 진짜 필요한가?** — 90% 이상의 경우 답은 No
3. **CDN 한 줄로 끝나는가?** — Yes면 도입 OK
4. **나중에 Next.js로 옮길 때 짐 되지 않는가?** — Yes면 보류

### 4.4 도입 검토가 필요한 시점에만 추가할 것들
- **폼 제출** → 6 Phase에서 Formspree·Google Forms·Resend API 중 결정
- **애널리틱스** → GA4, Meta Pixel, Clarity (Phase 6에서)
- **OG 이미지 자동 생성** → Phase 6 이후
- **다국어** → 검토 안 함

## 5. 콘텐츠 헌법

### 5.1 카피 톤
- **짧고 단호하게.** 한 문장 30자 이하 권장
- **숫자 우선.** "많은 분이"가 아니라 "5명", "100만 데이터"
- **수식어 절제.** "정말 놀라운", "엄청난" 같은 형용사 금지
- **존댓말 일관.** 반말·존댓말 섞임 금지
- **약속은 명확하게, 과장은 금물.** "첫 결제 받기 보장" ❌ → "첫 결제 시도까지 함께" ✓

### 5.2 핵심 카피 (변경 시 사업 컨셉 재검토 필요)
- 헤드: "4주 안에 AI로 당신의 첫 서비스를 출시합니다"
- 인사이트: "잘 만들어진 상품에 당신의 아이디어 20%를 섞으세요"
- 정체성: "AI 시대에 살아남는 사람은 좋은 기술자가 아닌 좋은 기획자입니다"
- CTA: "1기 무료 상담 신청"

### 5.3 신뢰 자산 (강사 신뢰 섹션 외에는 등장 금지)
- 크몽 자동매매 단건 430만원 판매
- BlogAutoFriends 운영
- PageForge, 캔들 릴스 생성기, 유튜브 자동화, Hermes
- 사업소득 연 5,000만원
- 비전공자 출신

이 자산들은 **[5] 강사 신뢰 섹션**에서만 등장. 다른 섹션에서 자랑처럼 흩어 박지 않는다.

### 5.4 콘텐츠 결정 기준
1. **이 문장이 1기 신청을 한 발 가깝게 하는가?** — No면 빼라
2. **이 문장이 신뢰를 무너뜨리지 않는가?** — 의심스러우면 빼라
3. **이 문장이 다른 섹션의 같은 메시지와 중복되는가?** — Yes면 둘 중 약한 쪽 제거

## 7. Phase 운영 원칙

### 7.1 Phase 분할
| Phase | 범위 | 산출물 |
|---|---|---|
| Phase 1 | 셋업 + 13섹션 골조 + 디자인 토큰 | `index.html` 골조, `style-guide.html` |
| Phase 2 | 히어로 + 마지막 CTA | 섹션 1, 13 완성 |
| Phase 3 | 강사 신뢰 + 페르소나 분석 시스템 | 섹션 5, 6 완성 |
| Phase 4 | 문제 공감 + 통찰 + 비교 | 섹션 2, 3, 4 완성 |
| Phase 5 | 커리큘럼 + 결과물 + 1:1 비교 | 섹션 7, 8, 9 완성 |
| Phase 6 | 대상 + 가격 + FAQ + 폴리싱 | 섹션 10, 11, 12 완성 + 폼·트래킹·OG |

### 7.2 Phase 운영 규칙
- **한 번에 한 Phase만 진행.** 동시 진행 금지
- **이전 Phase 완료 체크리스트가 다 통과돼야 다음 Phase 시작.**
- **Phase 중 발견된 이슈는 해당 Phase 내에서 해결.** 다음 Phase로 미루지 않음
- **Phase 종료 시 git commit 필수.** 메시지 형식: `feat: phase N - 섹션명`
- **Phase 종료 후 PROJECT.md 또는 README.md에 진행 상태 갱신**

### 7.3 Phase 외 작업 금지
- Phase 1 진행 중에 "기왕 하는 거 폼도 만들까?" → 금지
- Phase 3 진행 중에 "스크롤 애니메이션 넣을까?" → Phase 6에서 검토
- 의욕 폭주가 가장 큰 위험. 각 Phase는 자기 일만 한다.

## 8. 의사결정 우선순위

판단이 갈릴 때 다음 순서로 적용:

1. **1기 5명 모집에 도움이 되는가?** 
2. **사이트 톤(Linear·Stripe·Notion + YC)이 깨지지 않는가?**
3. **6개월 후 Next.js 이전 시 짐이 되지 않는가?**
4. **현재 Phase의 범위 내인가?**

위 4개 중 하나라도 명확히 No면 그 작업은 보류 또는 거부.

## 9. 성공 정의

이 프로젝트의 성공 = 다음 3가지 모두 달성:

1. **1기 모집 마감** (5명 결제 완료)
2. **상담 신청 전환율 측정 가능한 상태** (GA4·Meta Pixel·Clarity 정상 작동)
3. **2기 운영을 위한 데이터 확보** (어느 섹션에서 이탈하는지·어느 카피가 클릭을 끄는지)

이 3가지가 충족되면 6개월 뒤 통합 플랫폼 검토 단계로 넘어감.

---

## 10. 헌법 개정 절차

이 문서는 **함부로 바꾸지 않는다.** 다음 경우에만 개정:

1. 사업 모델이 근본적으로 바뀜 (예: 1:1 → 그룹 코호트)
2. 1기 운영 후 명확한 데이터로 방향 수정 필요
3. 디자인 톤 자체를 재정의해야 할 강력한 이유 발견

개정 시 절차:
1. 변경 사유를 README.md 또는 별도 파일에 기록
2. PROJECT.md 수정 + git commit 메시지 명확히 (`docs: amend constitution - 변경 사유`)
3. 변경 후 진행 중 Phase에 영향 검토

## 부록 A. 자주 흔들리는 결정에 대한 미리 답

**Q. 영상 콘텐츠 임베드해도 될까요?**  
A. Phase 6 이전엔 No. Phase 6에서 유튜브 1개만 [5] 강사 신뢰 섹션에 검토 가능.

**Q. 채팅 위젯(채널톡 등) 붙일까요?**  
A. 1기 5명 규모에서는 No. 신청 폼이 곧 채팅 진입점.

**Q. 다크모드 지원할까요?**  
A. No. 톤 일관성 깨짐.

**Q. 한국어 외 언어 지원?**  
A. No. 타겟이 한국 사용자 한정.

**Q. AB 테스트 도구 도입?**  
A. 1기 모집 트래픽으로는 의미 없음. 2기 이후 검토.

**Q. 후기·증언 섹션 추가?**
