### 핵심 학습 (이 흐름에서 얻은 결론)
1. **vw 단위 + 브라우저 줌 = 비율 깨짐** → Tailwind 기본 rem 기반 분기점이 가장 안정적 (commit `c401976`)
2. **em 단위 폰트 사이즈 의존도가 클수록 줌 시 일관성 좋음** → 그러나 element 간 비례 고정이 필요할 때만 (commit `0876c39`)
3. **letter-spacing(tracking)이 마지막 글자 뒤에도 적용** → text-center 정렬 시 어긋남 → `text-indent`로 좌측 보정 (commit `64ce3ea`)
4. **부모 박스가 자식 width 합산으로 결정될 때** absolute 자식이 부모보다 큰 컨텐츠를 담으면 외부로 흘러나옴 → `#hero` 직접 자식으로 분리 (commit `27489e5`)

## 7. 핵심 기술 결정 (정리)

| 영역 | 결정 | 이유 |
|---|---|---|
| 폰트 | Pretendard Variable (CDN) | 단일 폰트 + 가변 weight |
| CSS | Tailwind CDN + 인라인 `<style>` 미디어쿼리 | 빌드 없음, 임의값 클래스 보조 |
| 다크 톤 | `bg-black` / `bg-slate-900`(#0F172A) / `bg-[#111827]` | 시각적 깊이 단계 |
| 텍스트 사이즈 | Tailwind 기본 px 클래스 (text-* 분기) | 줌 안정성, em·vw 폐기 |
| 폼 백엔드 | Formspree (출시 시 키 발급) | 빌드 없는 정적 호스팅 |
| 트래킹 | GA4 + Meta Pixel + Microsoft Clarity (출시 시 키) | 무료, 라이트하게 |
| 호스팅 | Vercel 또는 GitHub Pages (출시 시 결정) | 정적 HTML, 자동 HTTPS |
| 도메인 | `amakers.co.kr` 권장 | 브랜드 일관성 |
